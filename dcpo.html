dcpo

Douglas Harris

Formally dcpo means directed-complete partially ordered set.

A set S is directed if every finite subset has an upper bound.

  • This goes a little farther than saying that there is no entity without any of their properties
  • It says there is an etity which actually rejects them all.

In a dcpo such a decision process will reach a result.

Of course the question now will be, how do you find a dcpo.

An ordered set X is a dcpo if every subset that is a directed ^-set has a lub.

  • Requiring an upper bound would just mean “there are entities”;
  • this requires “there is one and only one”.

Not too surprisingly there is a way to phrase this in our filter language. A directed set generates a nice kind of filter, called a UF for union filter.

FACT: O^p is a UF if every element meets the limit set of the filter.

  • This mean that the limit set actually belongs to the dual.
  • That is, every element is the limit of a NF which belongs to the dual.

You might want to compare

  • We have already learned that in a JF every element is a limit of a fJF which belongs to the dual.

  • Yes, these are NOT the same thing: there are fJF which are not NF.

In fact:

FACT: “every jJF is a NFi” iff “every JF is a UF”.

A space is called sober if every UF is a NF.

There are those who believe that we should only consider sober spaces.

(I am not one of those, because my favorite type of space, the poset, is not always sober).

FACT: A UF is prime, of course, since any union of PF is a PF (we already know a join (even of NF) may not be a PF).

A UF has an even stronger property. which is often called completely prime.

FACT: O^p is a UF iff whenever a member is a union one of the summands is already a member.

In other words taking unions of members does not give you anything new.

A PF was defined by that same statement, limited to joins (that is unions of finite sets of members).
